    Decimal System

    [Back to Contents]

  • Counts in 10's using the marks 0-9 to represent values.

    32 = 3x101 +2x100

    
4281 = 4x103 + 2x102 + 8x101+ 1x100

  • Each digit in the number has positional value and represents the number of occurrences of the weight of that position.

  • Non-integral numbers (real numbers)

    4281.82=4x103+2x102+8x101      +1x100 + 8x10-1+2x10-2

    Here, the decimal 'weights' are:

    103, 102,
101, 100, 10-1, 10-2

    Decimal to Binary Conversion of whole numbers

    [Back to Contents]

  • For whole numbers, divide by 2 and write result beneath with remainder to the side (1 or 0) and repeat process until fraction obtained.

    Example: Convert 41 to binary form.

    Fixed Point Storage of Binary Numbers

  • Consider a 16 bit word (for demonstration purposes only) where the binary point is taken to occur between the 8th and 9th entries. Thus,

    DecimalBinary Fixed Point Storage
    5.0101.00000010100000000
    5.5101.10000010110000000
    5.75101.110000010111000000
    5.875101.1110000010111100000

  • A problem occurs if number becomes so large that the number of bits required to represent it using fixed point storage exceeds the word length.

    Example:

    500.875111110100.11111111010011100000

    1 on LHS of number above gives 'Overflow'

  • If the number of bits required exceeds the word length, the result is called 'overflow'.

  • Word length sets a limit on the magnitude of the number that can be stored.

  • Integer decimal numbers have an exact representation in a binary number system.

    Integer arithmetic is exact.

  • Most real decimal numbers can not be represented exactly using a binary number system.

    Real arithmetic is not exact -- an important aspect of numerical computing.
